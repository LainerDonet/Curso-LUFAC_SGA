Vinculación de interfaz de red( Network Interface Bonding)

La Vinculación de interfaz de red es otra de las acciones que se pueden llevar a cabo desde LUFAC_SGA. La Vinculación de interfaz de red (Network Interface Bonding) (también conocida como Link Aggregation, LAGs, LACP o 802.3ad / 802.1ax) combina dos o más interfaces de red físicas en una única interfaz lógica. Estas son dos razones principales para usar el bonding de interfaces:

  1. Redundancia (balanceo de carga): Si hay un problema con un switch Ethernet, un puerto de switch o la conexión física, el tráfico se redirige automáticamente a otros puertos del switch. Esto proporciona una mayor disponibilidad de red en caso de fallos.
  2. Velocidad: Al combinar dos o más interfaces, se logra un balanceo de carga que aumenta el rendimiento general. Aunque hay un aumento en la sobrecarga, también se incrementa el ancho de banda disponible

El bonding de interfaces ofrece tanto redundancia como mayor velocidad. Es útil en entornos donde se necesita alta disponibilidad y mayor capacidad de transferencia de datos.
