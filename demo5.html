SystemD

SystemD es el administrador de servicios y sistemas en Linux, y se ha convertido en la estandarización en la mayoría de las distribuciones basadas en Debian y Red Hat. Fue desarrollado con el objetivo de encargarse de arrancar todo lo que está por debajo del Kernel, permitiendo ejecutar varios procesos de manera simultánea1. Algunas de sus características clave son:

  1. Paralelización: SystemD proporciona una notable capacidad de paralelización en el inicio de servicios.
  2. Activación de socket y D-Bus: Utiliza estos mecanismos para iniciar los servicios.
  3. Inicio bajo demanda: Los demonios se inician según la demanda.
  4. Seguimiento de procesos:Utiliza grupos de control de Linux para rastrear procesos.
  5. Snapshotting y restauración del estado del sistema:Permite crear instantáneas y restaurar el sistema.
  6. Gestión de dependencias lógica: Implementa un sistema de gestión de dependencias basado en el control lógico de los servicios.

Además, SystemD ofrece utilidades como systemctl para controlar el sistema y los servicios, journalctl para administrar el registro, hostnamectl para configurar el nombre de host, localectl para la distribución local y el teclado, y timedatectl para establecer la hora y la fecha del sistema

Objetivos(Targets)

Los .target en GNU/Linux son puntos de sincronización utilizados por el sistema para establecer un determinado estado. Estos targets están interrelacionados y permiten definir qué servicios y unidades se ejecutarán al iniciar el sistema operativo. Por ejemplo, el sysinit.target inicia todos los servicios y unidades de bajo nivel necesarios para que el sistema sea marginalmente funcional y permita el paso al target básico

Sockets

Los sockets en GNU/Linux son interfaces de entrada-salida de datos que permiten la intercomunicación entre procesos. Funcionan de manera similar a los teléfonos, permitiendo la comunicación entre procesos, ya sea en el mismo sistema o a través de una red1. Aquí tienes algunos detalles clave:

  1. Dominios de comunicación: Los sockets se crean dentro de un dominio de comunicación. Por ejemplo:
    • AF_INET: Para procesos en distintos sistemas unidos mediante una red TCP/IP.
    • AF_UNIX: Para procesos en el mismo sistema.
    • También existen otros dominios de comunicación1.
  2. Tipos de sockets en AF_INET:
    • Sockets Stream: Utilizan el protocolo TCP, proporcionando un flujo de datos bidireccional, secuenciado y libre de errores.
    • Sockets Datagram: Utilizan el protocolo UDP, con paquetes que pueden llegar fuera de secuencia o contener errores.

Los sockets son fundamentales para la comunicación entre procesos y permiten conexiones rápidas y eficientes sin sobrecarga de red

Temporizadores(Timers)

Los archivos .timer en GNU/Linux son unidades de SystemD que permiten programar tareas para que se ejecuten en momentos específicos o con intervalos regulares. Un archivo .timer es un archivo de configuración que define un temporizador. Puede estar asociado a una unidad de servicio o a una tarea específica. Los .timer se utilizan para activar servicios o tareas en momentos específicos, como iniciar una copia de seguridad diaria o ejecutar una limpieza semanal. Los archivos .timer se encuentran en el directorio /etc/systemd/system/ o /usr/lib/systemd/system/. Su sintaxis es similar a otros archivos de configuración de SystemD. Los archivos .timer son una herramienta poderosa para automatizar tareas en sistemas Linux mediante SystemD.

Rutas(Path)

La variable de entorno PATH en GNU/Linux es una lista de rutas que especifica dónde se encuentran los archivos ejecutables en el sistema. Cuando ejecutas un comando en la terminal, el intérprete de comandos busca esos ejecutables en los directorios listados en PATH en un orden específico. Por ejemplo, si deseas ejecutar un comando como ls , el sistema buscará en los directorios especificados en PATH para encontrar el archivo ejecutable correspondiente
