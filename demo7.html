Logical Volumen Manager(LVM)

Logical Volumen Manager es una herramienta que permite convertir una serie de discos o dispositivos de almacenamiento físicos en un volume group o grupo de volumenes y a su vez este grupo de volumenes para darle espacio a logical volume o volumenes lógicos. Que es mucho mejor user en el ambiente de servidores y clusteres, que las particiones comunes. En las particiones físicas si queriamos tener el home separado del root tenía que ser una partición física. Si no quedamos sin espacio era complicado hacer una migración. LVM surge para ayudarnos con este tipo de problemas. Podemos expadir las particiones.

Una de la ventajas de no se suele decir es que podemos tener diferentes LVM con diferentes file system. Ejemplo si tenemos una maquina de alto rendimiento que esta sirviendo mucha información; podemos tener un LVM /var/www la podemos tener como xfs y podemos tener el root como ext4. Pero no es la única ventaja LVM te permite usar diferente tecnología del almacenamiento dentro de un mismo volume group, por ejemplo: Puedes tener un SATA y un MDOS funcionando en conjunto, dondo la misma cantidad de espacio como si fuera un 2 en 1

LUFAC_SGA integra LVM2, una evolución de LVM1 con mejoras significativas en términos de eficiencia, flexibilidad y confiabilidad. LVM2 reemplazó a LVM1 y es compatible con LVM1, las mejoras que dispone son:

  • Marco de trabajo de kernel más genérico: LVM2 utiliza el controlador de mapeo de dispositivos (device mapper) del kernel 2.6.
  • Metadatos más eficientes: LVM2 utiliza un formato de metadatos más eficiente.
  • Recuperación mejorada: LVM2 tiene un mejor formato de recuperación.
  • Metadatos en formato ASCII: Facilita la administración y la recuperación.
  • Cambios atómicos en los metadatos: Evita problemas de consistencia.
  • Copias redundantes de los metadatos: Mayor fiabilidad.

Es posible convertir un grupo de volúmenes de LVM1 a LVM2 utilizando el comando vgconvert.
